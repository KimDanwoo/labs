# Commit Message Convention

프로젝트 커밋 메시지 규칙입니다.

---

# 기본 규칙

- 모든 커밋 메시지는 한글로 작성합니다.
- 커밋 메시지는 `type: 내용` 형식으로 작성합니다.
- 하나의 커밋은 하나의 목적만 포함합니다.
- 커밋 메시지만 보고 작업 내용을 이해할 수 있도록 작성합니다.
- 의미 없는 메시지(`update`, `fix`, `modify`)는 사용하지 않습니다.
- 커밋 메시지 끝에 마침표(`.`)를 사용하지 않습니다.

---

# Commit Types

| 타입 | 설명 |
| --- | --- |
| feat | 기능 추가 |
| fix | 버그 수정 |
| ref | 리팩토링 |
| chore | 빌드 및 환경 변경 |
| docs | 문서 수정 |
| test | 테스트 코드 수정 |
| style | 스타일 수정 (로직 변경 없음) |
| perf | 성능 개선 |
| ci | 지속적 통합 수정 |
| revert | 이전 커밋 되돌리기 |
| merge | 머지 커밋 |
| wip | 작업 중 |

---

# Commit Message Format

```bash
type: 작업 내용
```

## 예시

```bash
feat: 회원가입 API 연동
feat: 사주 계산 엔진 구축

fix: 로그인 토큰 갱신 오류 수정
fix: 사주 계산 엔진 버그 수정

ref: 유저 상태 관리 구조 개선
ref: 사주 계산 엔진 리팩토링

chore: eslint 설정 업데이트
chore: 사주 계산 엔진 빌드 및 환경 변경

docs: README 실행 방법 수정
docs: 사주 계산 엔진 문서 수정

test: 로그인 훅 테스트 코드 추가
test: 사주 계산 엔진 테스트 코드 수정

style: 버튼 컴포넌트 스타일 정리
style: 사주 계산 엔진 스타일 수정

perf: 이미지 lazy loading 적용
perf: 사주 계산 엔진 성능 개선

ci: github actions 배포 설정 수정
ci: 사주 계산 엔진 지속적 통합 수정

revert: 사용자 권한 변경 로직 되돌리기
revert: 사주 계산 엔진 이전 커밋 되돌리기

merge: develop 브랜치 병합

wip: 검색 필터 기능 작업 중
```

---

# Commit Unit Rules

## 기능

- 하나의 커밋은 하나의 기능만 포함합니다.
- 여러 기능을 한 커밋에 포함하지 않습니다.

### 좋은 예시

```bash
feat: 회원가입 API 연동
```

### 좋지 않은 예시

```bash
feat: 회원가입 및 로그인 기능 추가
```

---

## 버그 수정

- 하나의 커밋은 하나의 버그 수정만 포함합니다.

### 좋은 예시

```bash
fix: 모바일 뒤로가기 시 화면 프리징 수정
```

---

## 리팩토링

- 리팩토링과 기능 추가를 함께 포함하지 않습니다.
- 리팩토링과 버그 수정을 함께 포함하지 않습니다.

### 좋은 예시

```bash
ref: 인증 로직 구조 개선
```

---

## 스타일 수정

- 스타일 수정은 로직 변경 없이 UI 및 코드 스타일만 수정합니다.

### 좋은 예시

```bash
style: 버튼 간격 및 폰트 스타일 수정
```

---

## 문서 수정

- 문서 수정은 별도 커밋으로 분리합니다.

### 좋은 예시

```bash
docs: 설치 가이드 문서 추가
```

---

## 테스트 코드

- 테스트 코드는 가능하면 별도 커밋으로 분리합니다.

### 좋은 예시

```bash
test: 회원가입 API 테스트 코드 추가
```

---

# Additional Rules

- 작업 중 임시 저장 목적 커밋은 `wip` 타입을 사용합니다.
- 불필요하게 큰 커밋은 지양합니다.
- 관련 없는 변경사항을 하나의 커밋으로 묶지 않습니다.
- 영어보다 자연스러운 한글 표현을 우선합니다.

---

# Recommended Examples

```bash
feat: 매물 상세 페이지 SEO 메타 태그 추가
fix: iOS Safari 뒤로가기 시 스크롤 복원 오류 수정
ref: 공통 모달 상태 관리 구조 개선
perf: 이미지 preload 적용으로 LCP 개선
docs: 프로젝트 실행 방법 문서화
style: 공통 버튼 hover 스타일 수정
test: 로그인 폼 validation 테스트 추가
chore: pnpm workspace 설정 정리
ci: github actions 캐시 설정 추가
wip: 지도 클러스터링 기능 작업 중
```
