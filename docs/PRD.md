# 플레이브 다마고치 — Product Requirements Document

## 1. 개요

**프로젝트명:** 플레이브 다마고치 (Plave Damagotchi)
**플랫폼:** 웹 (모바일 우선 반응형)
**기술 스택:** Next.js 16, React 19, TypeScript, Tailwind CSS 4
**아트 스타일:** 8비트 픽셀아트 + 동화 그림체 하이브리드
**목표:** 플레이브 캐릭터를 육성하고, 미니게임으로 재화를 벌고, 다른 유저와 만남을 통해 하트를 채우는 다마고치 게임

---

## 3. 핵심 게임 루프

```
캐릭터 선택 → 육성(밥/똥/수면) → 미니게임(재화 획득) → 상점(밥 구매) → 만남의 방(하트 충전) → 하트 만충 → 아기 탄생
```

## 4. 기능 상세

### Phase 1 — MVP (우선 구현)

#### 4.1 캐릭터 선택 화면
- 5종 캐릭터를 보여주고 하나를 선택
- 선택 후 이름 지정 가능
- LocalStorage에 캐릭터 상태 저장

#### 4.2 메인 화면 (다마고치 룸)
- 선택한 캐릭터가 방 안에서 **혼자 돌아다니며 노는** 애니메이션
- 상단 상태바: 배고픔 게이지, 청결도 게이지, 하트 게이지, 레벨, 보유 재화
- 하단 액션 버튼: 밥주기, 똥치우기, 미니게임, 만남의 방

#### 4.3 밥 주기
- 보유한 음식 아이템으로 밥 주기
- 밥을 주면 **배고픔 게이지 회복**
- 시간이 지나면 배고픔 게이지 자동 감소 (1분당 1 감소, 최대 100)
- 밥을 너무 안 주면 (배고픔 0 → 일정 시간 경과) **캐릭터 사망**

#### 4.4 똥 싸기 / 치우기
- 일정 시간 간격으로 캐릭터가 **자동으로 똥을 쌈**
- 방에 똥이 쌓이면 **청결도 게이지 감소**
- 똥치우기 버튼으로 청소
- 너무 안 치우면 (청결도 0 → 일정 시간 경과) **캐릭터 사망**

#### 4.5 수면
- 밤 시간(실제 시간 기준 23시~7시)에는 캐릭터가 자동 수면
- 수면 중에는 게이지 감소 속도 절반

#### 4.6 사망 & 리셋
- 배고픔 0 또는 청결도 0 상태로 **3분** 이상 방치 시 사망
- 사망 시 슬픈 연출 후 캐릭터 선택 화면으로 돌아감
- 기존 재화는 유지

#### 4.7 만남의 방
- "만남의 방" 버튼 클릭 → 매칭 대기 화면
- **1:1 매칭**: 동시에 접속한 다른 유저 한 명과 만남
- 만남 성사 시 양쪽 캐릭터가 화면에 함께 표시
- 만남 완료 시 **하트 게이지 +10** 충전
- 하트 게이지 최대 100

#### 4.8 하트 시스템
- 만남을 통해서만 하트 충전 가능
- 하트 게이지가 100이 되면 **Phase 2 이벤트 트리거** (아기 탄생)

#### 4.9 레벨 시스템
- 경험치는 밥 주기, 똥 치우기, 미니게임, 만남 등 모든 활동에서 획득
- 레벨업 시 캐릭터 외형 약간 변화 (크기 증가 등)

---

### Phase 2 — 미니게임 & 확장

#### 4.10 미니게임: 바나나 캐치
- 하늘에서 바나나가 떨어지고 캐릭터를 좌우로 움직여 받기
- 폭탄/장애물 회피
- 점수에 따라 **코인(재화)** 획득
- 터치/마우스 드래그 조작

#### 4.11 미니게임: 플레이브 런 (쿠키런 스타일)
- 자동 달리기 + 점프/슬라이드로 장애물 회피
- 코인 수집
- 거리에 따라 추가 보상

#### 4.12 상점
- 미니게임에서 번 코인으로 음식 아이템 구매
- 음식 종류별 배고픔 회복량 차이
  - 🍞 빵 (10코인) → 배고픔 +10
  - 🍙 주먹밥 (20코인) → 배고픔 +25
  - 🍖 고기 (50코인) → 배고픔 +50
  - 🎂 케이크 (100코인) → 배고픔 +100 (완전 회복)

#### 4.13 아기 캐릭터 탄생
- 하트 게이지 100 도달 시 이벤트 발생
- "양갈래" 캐릭터가 아기로 탄생
- 아기 캐릭터는 메인 캐릭터와 함께 방에서 돌아다님
- 아기도 별도로 밥 주기/똥 치우기 필요 (난이도 상승)

## 5. 게이지 & 수치 밸런스

| 게이지 | 최대값 | 자동 감소 | 사망 조건 |
|--------|--------|-----------|-----------|
| 배고픔 | 100 | -1 / 분 | 0 유지 3분 |
| 청결도 | 100 | 똥 1개당 -10 | 0 유지 3분 |
| 하트 | 100 | 감소 없음 | — |

| 행동 | 경험치 | 비고 |
|------|--------|------|
| 밥 주기 | +5 | — |
| 똥 치우기 | +3 | — |
| 미니게임 클리어 | +10~30 | 점수 비례 |
| 만남 완료 | +15 | — |

| 레벨 | 필요 경험치 | 변화 |
|------|------------|------|
| 1→2 | 50 | 크기 약간 증가 |
| 2→3 | 150 | 표정 변화 추가 |
| 3→4 | 300 | 악세서리 등장 |
| 4→5 | 500 | 최종 진화 형태 |

## 6. 기술 아키텍처

### 6.1 프론트엔드 구조

```
app/
├── page.tsx                    # 캐릭터 선택 (홈)
├── layout.tsx                  # 글로벌 레이아웃
├── game/
│   └── page.tsx                # 메인 다마고치 룸
├── minigame/
│   ├── banana/page.tsx         # 바나나 캐치
│   └── run/page.tsx            # 플레이브 런
├── shop/
│   └── page.tsx                # 상점
├── meeting/
│   └── page.tsx                # 만남의 방
components/
├── character/
│   ├── CharacterSprite.tsx     # 캐릭터 렌더링
│   ├── CharacterSelect.tsx     # 캐릭터 선택 UI
│   └── CharacterAnimation.tsx  # 걸어다니기 애니메이션
├── game/
│   ├── StatusBar.tsx           # 상태 게이지 바
│   ├── ActionButtons.tsx       # 하단 액션 버튼
│   ├── Room.tsx                # 방 배경 + 캐릭터 영역
│   └── Poop.tsx                # 똥 오브젝트
├── minigame/
│   ├── BananaCatch.tsx         # 바나나 캐치 게임
│   └── PlaveRun.tsx            # 러닝 게임
├── shop/
│   └── ShopItem.tsx            # 상점 아이템
└── meeting/
    └── MeetingRoom.tsx         # 만남의 방 UI
hooks/
├── useGameState.ts             # 게임 상태 관리 (배고픔, 청결도, 하트, 레벨)
├── useAutoDecay.ts             # 게이지 자동 감소 타이머
├── useCharacterMovement.ts     # 캐릭터 랜덤 이동
└── useLocalStorage.ts          # 로컬스토리지 영속화
types/
└── game.ts                     # 게임 관련 타입 정의
constants/
└── game.ts                     # 게임 상수 (밸런스 수치 등)
```

### 6.2 상태 관리
- **게임 상태**: `useReducer` + LocalStorage 영속화
- **실시간 매칭 (만남의 방)**: WebSocket 또는 Supabase Realtime
- **게이지 자동 감소**: `setInterval` 기반 커스텀 훅

### 6.3 렌더링
- 캐릭터/배경: CSS 스프라이트 또는 Canvas 2D
- 미니게임: Canvas 2D (requestAnimationFrame 루프)
- UI: Tailwind CSS

### 6.4 데이터 영속화
- Phase 1: LocalStorage (오프라인 단독 플레이)
- Phase 2: Supabase (매칭, 랭킹 등 멀티플레이 기능)

## 7. 아트 가이드

### 스타일
- **8비트 픽셀아트** 기반, 동화적 색감
- 캐릭터: 32x32 ~ 64x64px 스프라이트
- 배경: 부드러운 파스텔톤 방 배경
- UI: 둥근 픽셀 폰트, 부드러운 버튼

### 캐릭터 스프라이트 필요 목록
- 각 캐릭터별:
  - idle (서있기) — 2프레임
  - walk (걷기) — 4프레임
  - eat (먹기) — 3프레임
  - sleep (수면) — 2프레임
  - happy (기쁨) — 2프레임
  - sad (슬픔) — 2프레임
  - dead (사망) — 1프레임
- 오브젝트:
  - 똥 — 1프레임
  - 음식 아이템 4종
  - 하트 이펙트
  - 코인

## 8. 구현 우선순위

### 🟢 Phase 1 — MVP (현재 스프린트)
1. 캐릭터 선택 화면
2. 메인 다마고치 룸 (캐릭터 돌아다니기)
3. 밥 주기 기능
4. 똥 싸기/치우기 기능
5. 게이지 시스템 (배고픔, 청결도)
6. 사망/리셋 시스템
7. 만남의 방 (로컬 시뮬레이션 — 실제 매칭은 Phase 2)
8. 하트 시스템

### 🟡 Phase 2 — 미니게임 & 소셜
1. 바나나 캐치 미니게임
2. 플레이브 런 미니게임
3. 상점 시스템
4. 실시간 매칭 (Supabase)
5. 아기 캐릭터 탄생

### 🔴 Phase 3 — 폴리싱
1. 사운드 이펙트 & BGM
2. 레벨업 연출
3. 랭킹 시스템
4. 캐릭터 커스터마이징 (악세서리)
5. PWA 지원 (오프라인 & 푸시 알림)
